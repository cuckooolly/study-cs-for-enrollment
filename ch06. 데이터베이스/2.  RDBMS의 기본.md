# RDBMS의 기본
## 테이블의 구성: 필드와 레코드
- 레코드: 행(row), 튜플(tuple)
- 필드: 열(column), 애트리뷰트(attribute)

### 필드 타입

- RDBMS 마다 다른 타입을 지원하므로, 사용하는 RDBMS의 문서를 참고

### 키

- 키: 테이블의 레코드를 식별할 수 있는 하나 이상의 필드
  - 후보 키: 테이블의 한 레코드를 유일하게 식별할 수 있는 필드의 최소 집합 (유일성과 최소성 만족)
  - 기본 키: 테이블의 각 레코드를 유일하게 식별할 수 있는 필드로, 테이블 당 하나만 존재
  - 외래 키: 다른 테이블의 기본 키를 참조하는 필드로, 테이블 간의 관계를 정의

## 테이블의 관계

### 일대일 대응 관계

- 하나의 레코드가 다른 테이블의 레코드 하나에만 대응되는 관계

![](https://csnote.net/assets/img/db/onetoone.png)
### 일대다 대응 관계

- 하나의 레코드가 다른 테이블의 여러 레코드와 대응되는 관계

![](https://csnote.net/assets/img/db/onetomany.png)

### 다대다 대응 관계

- 여러 레코드가 다른 테이블의 여러 레코드와 연결되는 관계
![](https://csnote.net/assets/img/db/manytomany.png)

## 무결성 제약 조건

- 데이터베이스의 일관성과 유효한 데이터를 유지하기 위해 지켜야 하는 규칙

### 도메인 제약 조건
- 테이블이 가질 수 있는 필드 타입과 범위에 대한 규칙
- 원자 값: 더 이상 쪼갤 수 없는 단일한 값
### 키 제약 조건
- 키로 지정된 필드에 중복된 값이 존재해서는 안 되는 제약 조건
### 엔티티 무결성 제약 조건
- 기본 키로 지정된 필드는 고유한 값이여야 하며, NULL이 되어서는 안되는 규칙
### 참조 무결성 제약 조건
- 외래 키를 통해 참조된 데이터는 실제로 존재해야 하는 규칙

- 참조하는 테이블이 삭제/수정되는 경우
  - 연산 제한: 주어진 수정 및 삭제 연산 자체를 거부
  - 기본값 설정: 참조하는 레코드를 미리 지정한 기본값으로 설정
  - NULL 값 설정: 참조하는 레코드를 NULL로 설정
  - 연쇄 변경: 참조하는 레코드도 함께 수정되거나 삭제
