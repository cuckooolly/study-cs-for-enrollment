# 네트워크의 큰 그림

## 네트워크의 기본 구조

- 네트워크는 그래프로 표현할 수 있음
![](https://csnote.net/assets/img/net/network.png)

- 네트워크 토폴로지: 네트워크 상에서 노드와 노드 사이의 연결 구조
![](https://csnote.net/assets/img/net/network_topology.png)

- 호스트: 네트워크에서 가장자리 노드로, 종단 시스템(end system)이라고도 부름
  - 서버: 서비스를 제공하는 호스트로, 요청 메세지를 받고 응답 메시지를 보냄
  - 클라이언트: 서버에게 어떠한 서비스를 요청하고 서버의 응답을 제공받는 호스트

### LAN과 WAN
- LAN(Local Area Network): 근거리 통신망
- WAN(Wide Area Network): 광역 통신망
- MAN(Metropolitan Area Network): 도시 통신망
![](https://csnote.net/assets/img/net/lan.png)

### 패킷 교환 네트워크
- 패킷 교환 네트워크: 데이터를 작은 단위인 패킷으로 나누어 전송하는 네트워크
- 회선 교환 네트워크: 통신을 위해 송신자와 수신자 사이에 전용 회선을 설정하는 네트워크

### 주소의 개념과 전송 방식
- 주소: 패킷의 헤더에 명시되는 정보, IP주소와 MAC 주소가 있음
- 다양한 메시지 송수신 방식
  - 유니캐스트: 송신지와 수신지가 1:1로 통신
  - 멀티캐스트: 송신지 1개가 여러 수신지에게 통신
  - 브로드캐스트: 송신지 1개가 네트워크에 연결된 모든 호스트에게 통신
  - 애니캐스트: 송신지 1개가 여러 수신지 중 가장 가까운 1개에게 통신

## 두 호스트가 패킷을 주고 받는 과정

### 프토로콜

- 프로토콜: 네트워크에서 통신하기 위한 규칙과 절차를 정의한 것
- 프로토콜 스택: 여러 프로토콜이 계층적으로 쌓여있는 구조
- 주요 프로토콜
  - HTTP/HTTPS: 웹 페이지를 주고 받기 위한 프로토콜
  - FTP: 파일 전송을 위한 프로토콜
  - SMTP/IMAP/POP3: 이메일 송수신을 위한 프로토콜
  - TCP: 신뢰성 있는 데이터 전송을 위한 프로토콜
  - UDP: 비신뢰성 데이터 전송을 위한 프로토콜
  - IP: 패킷의 주소 지정과 라우팅을 담당하는 프로토콜
  - Ethernet: LAN에서 데이터 링크 계층을 담당하는 프로토콜

### 네트워크 참조 모델

- 네트워크 참조 모델: 통신이 일어나는 구조를 계층으로 표현한 모델

#### OSI 7계층 모델

![](https://cdn.networkacademy.io/sites/default/files/2025-07/the-osi-model-overview.svg)

- 물리계층: 전기적 신호로 데이터를 전송하는 계층
- 데이터 링크 계층: 네트워크 내 주변 장치 간의 정보를 올바르게 주고받기 위한 계층
- 네트워크 계층: 패킷의 주소 지정과 라우팅을 담당하는 계층
- 전송 계층: 종단 간의 신뢰성 있는 데이터 전송을 담당하는 계층
- 세션 계층: 통신 세션을 설정, 관리, 종료하는 계층
- 표현 계층: 데이터의 형식과 인코딩을 담당하는 계층
- 응용 계층: 사용자가 이용하는 응용프로그램과 맞닿아 있는 계층, 네트워크 서비스가 제공되는 계층

#### TCP/IP 4계층 모델

![](https://cdn.networkacademy.io/sites/default/files/2025-07/osi-vs-tcp-ip-models.svg)

- 네트워크 엑세스 계층: 물리계층과 데이터 링크 계층을 포함하는 계층
- 인터넷 계층: 네트워크 계층에 해당하는 계층
- 전송 계층: OSI 모델과 동일하게 종단 간의 신뢰성 있는 데이터 전송을 담당하는 계층
- 응용 계층: OSI 모델의 세션, 표현, 응용 계층을 포함하는 계층

### 캡슐화와 역캡슐화

![](https://cdn.networkacademy.io/sites/default/files/2025-07/what-is-the-osi-model.svg)

- 캡슐화: 데이터 전송 과정에서 헤더 및 페이로드를 추가해 나가는 과정
- 역캡슐화: 캡슐화 과정에서 붙인 헤더 및 페이로드를 제거하는 과정
